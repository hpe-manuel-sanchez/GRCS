Amending Rights Data sets for Resources/Tracks

Only users with Edit Rights data (Header), Edit Rights data (Digital),Edit Rights data (Secondary), Edit Rights data (Pre-Cleared), Edit Rights Review Status privileges can search, view and Amend the rights data created for a resource/track in GRS. Using the Priority Rights Review WQ  user can review the rights/amend for required resources/tracks.

  1. Select Rights Administration -> Rights Review Work Queue. The Rights Review Filter accordion is displayed. By default, the ‘Rights Review Filter’ section is displayed. You can enter one or a combination of criteria to search for the resource (s)/ track (s).

  1. User can review the Rights  Data Sets in 4 Tabs:

    • Rights Acquired  Tab

    • Digital Restrictions Tab

    • Secondary Exploitation Tab

    • Pre- Clearance Tab

  1. By default, the Rights Acquired Tab is selected displaying the Resource(s)/Tracks with New- For- Review and Final- for – Review Status

  2. The below attributes are included in the display of the Work Queue grid for each Work queue item per row which can be amended individually on the Work queue grid.

Rights Acquired Tab

      • Active for Marketing flag:-You can update the value to ‘Yes/No’

      • Rights Period:- You click on individual row item(s) and the following values can be edited to

        • In Perpetuity

        • Life of Copyright

        • Other Duration

      • Lost Rights:- You can update the value to ‘Yes/No.

      • Lost Rights Date:- You click on individual row item and then the calendar value can be edited

      • Lost Rights Reason:- You click on individual row item(s) and the following values can be edited to

        • Rights Expiry

        • Rights Reversion

        • Litigation

      • Rights Exception:- You click on individual row item and the following values can be edited to

        • Exception Applied

        • Exception Not Applied

      • Physical Exploitation Rights flag:- You can update the value to ‘Yes/No'

      • Digital Exploitation Rights Flag:- You can update the value to ‘Yes/No’

      • Digital Unbundling Allowed:- You can update the value to ‘Yes/No’

      • Mobile Exploitation Rights flag:- You can update the value to ‘Yes/No’

      • PPB Revenue Claim:- You can update the value to ‘Yes/No

Digital Restriction Tab

 

  1. The below attributes are included in the display of the Work Queue grid for each Work queue item per row which can be amended individually on the Work queue grid.

      • Use Type:- You click on individual row item and the following values can be edited to

        • Download

        • Streaming

      • Commercial Models:- You click on individual row item and the following values can be edited to

        • A la carte

        • Subscription

        • Ad-Funded

        • All

      • Restrictions:-You click on individual row item and the following values can be edited to

        • No Rights

        • Consent Required

        • Refer to legal

        • No Restrictions

        • Consult

      • Restriction Reason:- You click on individual row item and the following values can be edited to

        • Artist Consent Required

        • Sample Issues

        • Side Artist Issues

        • Missing “No Rights”

        • Other

      • Consent Period:- You click on individual row item and the following values can be edited to

        • During Hold back Period

        • During Term

        • During & After Term

Notes:- User clicks on individual row item and the text can be edited

  1. The below attributes are included in the display of the Work Queue grid for each Work queue item per row which can be amended individually on the Work queue grid.
