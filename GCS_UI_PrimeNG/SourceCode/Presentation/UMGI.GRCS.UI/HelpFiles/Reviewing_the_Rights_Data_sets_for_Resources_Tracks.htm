Reviewing the Rights Data sets for Resources/Tracks

Only users with Edit Rights data (Header), Edit Rights data (Digital),Edit Rights data (Secondary), Edit Rights data (Pre-Cleared), Edit Rights Review Status privileges can search, view and Amend the rights data created for a resource/track in GRS. Using the Priority Rights Review WQ  user can review the rights/amend for required resources/tracks. You can click the following accordions to enter the respective fields for reviewing the rights of the required resources.

  • Rights Review Filter

  • Digital Restrictions

  1. Select Rights Administration -> Rights Review Work Queue. The Rights Review Filter accordion is displayed. By default, the ‘Rights Review Filter’ section is displayed. You can enter one or a combination of criteria to search for the resource (s)/ track (s).

  1. The Following Tabs can be accessed to Review/Amend the Rights for required Resource (s) / Track (s)

    • Rights Acquired Tab

    • Digital Restriction Tab

    • Secondary Exploitation Tab

    • Pre- Clearance Tab

  1. By default, the Rights Acquired Tab is selected displaying the Resource (s)/Track (s) with New- For- Review and Final- for – Review Status

Rights Acquired Tab

Rights Review Filter

 

  1. Select the attributes check box from Resource/Tracks Rights Data, Resource Types, Review Status, Sample Exists?, Side Artists Exists?  From the Accordion menu of Rights Review Filter and click Apply Filter. The Resource (s)/ Track (s) are filtered and displayed in the Rights Acquired Tab

Note:Rights Review Filter is common for all the 4 tabs

  1. You can also search for specific Resource (s) / Track (s) using the following search criteria fields:

    • Artist Name

    • Clearance Admin Company

    • Title

    • Linked Contract

    • ISRC

    • UPC

    • Reason for Review

Note:The above filters are common to all the 4 tabs

  1. Click Reset- All the search criteria entered will be cleared, ready for entering new search criteria.

  2. The list of resources matching the search criteria is displayed as a table on screen. Each row displays the following Resource Rights details:

    • Traits

    • ISRC

Note:System displays the ISRC as a link, on clicking on the ISRC ID user navigates to the Custom Release Rights Review Work Queue containing all the releases the ISRC is part of.

    • UPC

Note:System displays the UPCs only for track data items. It is identified as a multi-artist release or otherwise as a normal release. On clicking the UPC the user is navigated to the custom release rights review WQ for that UPC

    • Artist

    • Title

    • Version Title

    • Release Date- Earliest release date across all releases which resource is associated to

    • Linked Contract

      • Shown as a linked contract icon in grid, with a tool tip as- Concatenation of Artist or Contracting Party if no Artist, and Contract Effective Date.

      • When the linked contract is a Split contract, then icon for split contract is displayed with a tool tip same as above.

    • Clearance Admin Company

    • Side Artists Exists

    • Sample Exists

    • P Year

    • Rights Type

    • Reason For Review

    • Territorial Rights

      • If the length of countries exceeds those maximum characters, then you can point the cursor on the country to view the complete TRs in a tool tip.

    • Error

    • Active for Marketing

    • Rights Period

    • Lost Rights

    • Lost Rights Reason

    • Lost Rights Date

    • Rights Exception

    • Physical Exploitation Rights

    • Digital Exploitation Rights

    • Mobile Exploitation Rights

    • PPB Revenue Claim
