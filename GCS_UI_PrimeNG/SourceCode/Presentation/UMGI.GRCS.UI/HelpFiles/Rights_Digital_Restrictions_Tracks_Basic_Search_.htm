Rights & Digital Restrictions Tracks Basic Search

Only users with View Repertoire Rights/View Repertoire Rights Sensitive data’  privileges can search and view the rights data created for a release/resource/track in GRS. Using the Rights Tracks Basic Search, user can review the rights for required tracks. You can click the following accordions to enter the respective fields for reviewing the rights of the required Tracks:

  • Resources Parameters

  • Track’s Rights Parameters

  • Digital Restrictions

  1. Select Repertoire Rights Search -> Rights & Digital Restrictions -> Show Tracks -> Basic Search. The Rights & Digital Restrictions Tracks Basic Search screen is displayed.

  1. By default, the ‘Resources Parameters’ section is displayed. You can enter or select relevant information as search     criteria from the Resources parameters section.

Resource's Parameters

  1. Also, you can specify Resources Rights parameters as search criteria by entering or selecting information for the fields in the Track’s Rights Parameters section.

Track's Rights Parameters

  4. You can enter or select relevant information as search criteria in the Digital Restrictions section.

Digital Restrictions

  1. Use Type – Select a use type from the drop-down

  2. Commercial Models – Select a option from the drop-down

  3. Restrictions Exists – Select this check box if digital restrictions are available.

  4. Restrictions Doesn’t Exists - Select this check box if digital restrictions are not available.

  5. To add an individual Digital Restriction row, click the Add Restrictions hyper link. The Digital Restriction row is displayed.

  6. To remove a Digital Restriction row, click the Remove icon corresponding to the row you wish to remove.

  7. Click Search. The Tracks Rights & Digital Restrictions repertoire results that match the search criteria specified are displayed as list in the Search Results grid. The search Results grid is displayed as Paged format.

  1. The list of resources matching the search criteria is displayed as a table on screen. Each row displays the following Resource Rights details:

    • Traits

    • ISRC

    • UPC

    • Artist

    • Title

    • Version Title

    • Release Date- Earliest release date across all releases which resource is associated to

    • Linked Contract

      1. Shown as a linked contract icon in grid, with a tool tip as- Concatenation of Artist or Contracting Party if no Artist, and Contract Effective Date.

      2. When the linked contract is a Split contract, then icon for split contract is displayed with a tool tip same as above.

    • Clearance Admin Company

    • P Year

    • Review Status

    • Active for Marketing

    • Territorial Rights

      1. If the length of countries exceeds those maximum characters, then you can point the cursor on the country to view the complete TRs in a tool tip.

    • Rights Period

    • Lost Rights

    • Lost Rights Reason

    • Lost Rights Date

    • Physical Exploitation Rights

    • Digital Exploitation Rights

    • Mobile Exploitation Rights

    • PPB Revenue Claim

    • Digital Unbundling Allowed

    • Use Type

    • Commercial

    • Consent Period

    • Notes

  1. You select one or more rows from the search results and clicks on Review Rights Button then system ports the selected item (s) to the Custom Resource/Tracks Rights Review Work Queue.
