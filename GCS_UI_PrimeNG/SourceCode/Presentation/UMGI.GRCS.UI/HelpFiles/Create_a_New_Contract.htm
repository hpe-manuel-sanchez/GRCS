Create a New Contract

Contract Header    Territorial Rights    Repertoire Defaults    Clearance Controls    Notes & Notifications

  1. Login to GRS and the home page of the system is displayed.


      

  1. Select Contract Administration -> Create New Contract from the header section drop-down. The Create New Contract screen is displayed with three tabs. By default, the Contract tab is displayed. You can click the ‘Rights & Restrictions’ or ‘Secondary Exploitation’ tabs to access the respective fields and set further restrictions.


      

  1. Contract ID - Once you complete the mandatory fields and click Save, the system will generate a Contract ID number - this displays automatically in the top left corner of the screen. The contract ID is a unique, read only 11 digit number.

  1. Workflow Status - By default, the Contract Workflow Status is ‘Data Entry’ when the Contract entry screen first opened. The status can be changed to mark contracts as requiring approval (Pending Approval) or approved (Approved).

  2. Parent Contract - To link the new contract as a child contract to another existing contract, click the Assign Parent Contract hyperlink. The Search Parent Contract pop-up is displayed. (if no Parent Contract is required, skip to step 12).


      

  1. Enter search criteria in the required field(s), and click Search.

  2. The list of contracts matching the search criteria is displayed as a table on screen. Each row displays the following contract details: Artist, Contracting Party, Workflow Status, Contract Status, Contract Description, Clearance Admin Company, and Signing Company.

  3. To assign a parent contract, select the corresponding row of the contract in the table. A message is displayed asking if you wish to propagate the rights from the parent contract to your new contract.


            

  1. If you click Yes, the pop-up window is closed and the selected parent contract is displayed in the Parent Contract section. The rights data from the parent contract is also automatically copied onto the new contract. You can continue to edit the Child Contract rights data as necessary.

NoteThe Workflow Status of the new contract will be defaulted to ‘Data Entry’, regardless of the Parent contract status.:

  1. If you click No, then the Parent Contract is still linked and displayed in the Parent Contract section, but the rights data is not copied from the Parent Contract to the new Child Contract. You can continue entering details for the new contract.

  2. If required, you can remove the assigned parent contract by clicking the Unlink hyperlink in the action field. The unlinking action will just break the link between the parent and child contracts, and there will be no further impact on the new contract.

  3. Split Deal – Contracts that have been identified as Split Deals during the contract entry process will be displayed in this section. For further information on creating Split Deals see section 52 (hyperlink to section)

Contract Header

  1. Clearance Admin Company - Enter a Clearance Admin Company-Country by starting to write the company name in the textbox; the system then automatically searches and display a list of Clearance Admin Company-Country values that match the text entered.

NoteThe list only displays companies and countries that are defined in a user’s security profile.:

  1. Select an admin company from the list and the selected company is displayed in the text box. The list of Clearance Admin Company-Countries is pre-defined in the system, with data derived from Global Tables.

  2. UMG Signing Company - by default, the Clearance Admin Company-Country selected is also displayed as the UMG Signing Company. To modify the UMG signing company, write the desired company name in the textbox, and then select from the list of available signing companies from the auto-suggestion.

  3. Artist - To link an Artist to the contract, click the Add button adjacent to the Artist display field.  The Search for Artist pop-up page is then displayed.

NoteIf the Clearance Admin Company is not already selected, then the system displays a message prompting you to select a Clearance Admin Company first.:


      

  1. Enter either the Artist Name, First Name, Last/Group Name, or Talent ID in the respective search criteria fields and press Search.

  • Artist – this search works as a keyword search, searching for the text entered across First and Last/Group Name fields.

  • Search All Name Fields – ticking this box means the Artist search will extend across all Artist fields (including Additional Info & Display Names)

  • First Name – an exact match search on only the first name field

  • Last/Group Name – an exact match search on only the Last/Group Name field

  • Talent ID – an exact match search on the R2 talent ID field

NoteArtists details are provided via the R2 Interface.:

Select the artist by selecting the corresponding row from the table. The Search for Artist pop-up window is closed, and you are returned to the contract. The artist name and ID are displayed in their respective fields.

  1. Talent ID – this is a read-only field which displays the associated R2 talent ID once you have selected an Artist from the look-up.

  2. Contracting Party - If required, enter a Contracting Party. This can either be used instead of an artist name, or in addition to an artist name on the contract.

  3. Artist Name in Local Characters – This is an optional free text field to allow you to enter a different local name for an artist.

  4. Local Contract Reference Number – This is an optional free text field to allow you to enter a local reference number for the contract.

  5. Contract Status – select either the ‘Signed Contract’ or ‘Deal Memo/Draft Contract’ option from the Contract Status drop-down.

  6. Contract Description - select any description from the drop-down, or enter free text to describe the contract. This field accepts all alphanumeric and special characters. This is a mandatory field before a contract can be approved.

  7. Contract Commencement – select a date from the calendar lookup - or type in the date using the 01 Jan 2000 format. This is a mandatory field before a contract can be approved.

  8. End Of Term - select a date from the calendar lookup - or type in the date using the 01 Jan 2000 format. This is an optional field.

  9. On Active Roster - Select ‘Yes-Active’ or ‘No-Inactive’ from the On Active Roster drop-down as required.

Notes:

  1. If the end of term date is later than the current date, GRS automatically sets this field to ‘Yes-Active’.

  2. If the end of term date is set earlier than the current system date then GRS automatically sets the option as ‘No-Inactive’.

Territorial Rights & Rights Period

  1. To add territorial rights to the contract, click the Add/Edit Territorial Rights hyperlink:


      

The Territorial Rights pop-up window is displayed.


      
      

  1. In this pop-up, the countries are categorized and displayed hierarchically. For examples Universe -> World -> Territory -> Countries. By default, the countries and territories are displayed in a collapsed view under each territory grouping. Each section can be expanded by clicking the arrow alongside the header.

  2. The Territorial Rights pop-up displays three columns: Include, Exclude and N/A. By default, the list of countries and territories are all selected as N/A.

  3. You can search for a specific country or territory in the Auto Suggest search box. Enter the name of the Country/Territory required and click Find. The country or territory that matches the search criteria is displayed along with its hierarchy. For example, if you search for the country ‘Algeria’, then the hierarchy will be expanded under Universe -> World -> Africa -> Algeria.

Notes:

  1. If you select a country which is present in two territories, then the country is displayed in a row under each related territory.

  2. If you click the Clear Filter hyperlink, then the searched country/territory is removed and the original collapsed view is displayed.

  1. To add country/territory rights, select the Include radio button against the country/territory. The selected country/territory is then displayed in the Include text box.

  2. To remove country/territory rights, select the Exclude radio button. The selected country/territory is then displayed in the Exclude text box.

Note:

Type your drop-down text here.


      

  1. Once your selections are complete, click Save. The Territory Rights pop-up is then closed and you are returned to the Contract page. The selected country/territory is displayed as a concatenated statement in the Territorial Rights section.


      

  1. Rights Period - Set the contractual Rights Period using the available drop-down values. The Rights Period is set to Perpetuity by default. If you select “Other Duration”, the Rights Expiry Rule field becomes available for editing.

  2. Release Commitment & Rights Reversion is an optional free text field that can be used to enter relevant details.

  3. Rights Expiry Rule – a free text can be added in this field, but only if the Rights Period is set to “Other Duration”.

  4. Lost Rights Indicator – the drop-down is set to ‘No’ by default. If this is changed to ‘Yes’, you are presented with a warning & the contract will be clearly marked with a lost rights indication. If this field is set to Yes, a Lost Rights Reason is mandatory.

  5. Lost Rights Date - is an optional field, if the Lost Rights indicator is set to Yes. Set the date from the calendar lookup - or type in the date using the 01 Jan 2000 format.

  6. Lost Rights Reason - is mandatory if the Lost Rights drop-down is set to Yes. Select the reason from the available options in the drop-down.

  7. Rights Exceptions - a free text field that can be used to enter details of any contractual exceptions that need to be applied. It can contain any alpha-numerical values.

  8. Rights Exception Applied – this check box should only be set by a Legal Reviewer, after any exceptions have been applied to the repertoire linked to a contract.

Repertoire Defaults

  1. P/C Notice Company-Country - select a P/C Company from the drop-down list, or click Search to search the P/C company database.

NoteYour most frequently entered P/C Notice Companies-Countries are displayed in the drop-down.:

  1. If you click Search, the Search P/C Notice Company pop-up is displayed.

  2. Either enter a Country Name in the look-up field and/or enter a P-Company name in the Company Name search text box.

  3. Click Search and all P/C Notice Companies-Countries that match the search criteria are displayed in the Search Result grid.


      

  1. Select a P/C Notice Company-Country by selecting the corresponding row. The Search P/C Notice Company pop-up is closed and you are returned to the Contract page. The selected company is then displayed in the field.

  2. P/C Notice Company Extension - is a free text field that can be used to add extension text as required.

  3. Rights Type – select a value from the drop-down list. The value is set to ‘Owned By UMG’ by default.

  4. Legal Rights Review Required - Select ‘Yes’ or ‘No’ from the drop-down. This determines whether rights assigned to linked repertoire from this contract require legal review prior to finalisation (subject to review conditions defined by the owning label).

Notes:

  1. If the selected rights type is either ‘Owned by UMG’, ‘Exclusive License’, or ‘Joint Venture’, the system automatically sets ‘Yes’ for Legal Rights Review Required. If required, you can change it to ‘No’.

  2. If the selected rights type is ‘Non-UMG’, the system automatically sets ‘No’ for Legal Rights Review Required. If required, you can change it to ‘Yes’.

Clearance Controls

  1. Active for Marketing? – select Yes or No from the drop-down selection. By default it is set to Yes.

NoteThe system defaults the Active for Marketing field based on the On Active Roster field. If required, you can change the Active for Marketing option.:

  1. Sensitive Artist – select ‘Yes’ or ‘No’ from the drop-down.

  2. Clearance Notes – a free text field which can be used for additional clearance information.  

Notes & Notifications

  1. Email Notification Recipients – these are added by clicking the Address Book hyperlink, and selecting an existing group.

  2. Clicking the Address book link opens the Global Address List pop-up.


      

  1. Enter the required role/group name of the recipients in the text box and/or enter the country name in the Auto suggest Country Search field.

  2. Click Search and all Recipient Groups that match the search criteria are listed in the Search Result grid. The Global Address book Groups can be created and managed directly in GRS. For more information, refer to the ‘Manage Address Book’ section. (Hyperlink required)

  3. Select one or more recipient groups from the Search Result grid and click OK. The pop-up is closed and you are returned to the Contract page. The selected recipient groups are listed in the field separated by comma.

  4. General Notes - can be added in the text box if required. The text box accepts all alphanumeric and special characters.
