Reviewer Inbox

*You should be assigned to at least one workgroup with the reviewer (local label/UMGI Marketing/UMGI global Clearance/Marketing/Legal) role in the Clearance system

*Only Submitted/Resubmitted projects will be displayed in the Inbox of Reviewer

1. Login to GCS and the home page of the system is displayed.

Note: Home page of GCS system is Inbox view.

2. If you are a part of Reviewer as well as Requestor role, System displays 2 tabs on the home page, for Reviewer and for Requestor. You can view the default Inbox based on the settings done on User Preferences screen.

3. Similarly, if you are a part of RCC Admin role, system displays RCC Admin Inbox as default. Click on Reviewer tab to view and perform reviewer Inbox actions.

4. System displays the Reviewer’s Inbox with all the submitted/resubmitted projects based on routing.

5. Inbox is divided into two sections: Left hand side and Right hand side. You can drag the divider between two sections to increase/decrease the view of each section.

Left Hand Side section of Reviewer’s Inbox

1. Grouping of Projects: System groups the project in the following folders for Reviewer’s inbox:

  • High Priority : All the projects having high priority flag are listed under this folder.

  • Pending : All the non high priority projects are grouped under pending folder

  • Artist Consent : The projects where requests are marked for artist consent are grouped under artist consent folder

  • User created folder : These are personalized user folders

2. Default Folder: Whenever you log in to the Global Clearance System, by default, system displays the High Priority folder as expanded mode. If the high priority folder is empty, the system opens the pending folder by default.

Note: Only one folder can be in expanded mode at a time. If the user clicks on any other folder, the previously opened folder will become collapsed.

3. By default, System displays 20 requests in each folder. If the folder contains more than 20 requests, the system enables the Show All link. You can click on Show All link to view all the projects within the folder.

Note: The clearance system changes the Show All link to Show Default link to revert to the default mode.

4. All the projects in each folder will appear in bold when a notification has been received for the request. Once the requestor has clicked on a bold project, the bold highlight will be removed.

Note: The requestor can click on the “eye” icon to again highlight the project as bold to indicate unread.

5. System shows the following columns in the sequence listed below for each project.

  • Project Type

  • Project Reference Number

  • Project Title

  • Notification Date

  • Assigned To

  • Requestor Name

  • Release Date

  • Estimated Sales

  • Role

  • Workgroup

6. Click on Project Reference Number hyperlink, project opens in the new browser tab.

Note: By default, Project information tab is opened in non-editable mode.

7. Assigned To: You can assign the project pending for review by selecting one of the users from drop down list.

8. The system displays all the users of the workgroup with which request is pending. You can select one user from the drop down and click on Save icon next to the folder name. System will save the assigned to user name against the project and all the users who are a part of the same workgroup can view the name of the user to whom the request is assigned.

Note: If you are a part of more than one reviewer role, the same project may be pending in your inbox more than once if the requests have been routed to different roles. Adding assigned to user for one role will not have any impact on the other workgroup role for which the project is pending. However, if request is marked for artist consent from the same role for which assigned to user is being added/updated, the system will update it in artist consent folder as well and vice versa.

9. Filters: By clicking on Filters hyperlink, system opens the filters popup with the below given sections:

  • Request Type Filter: You can select one or more request types to filter the projects in Inbox. Only those projects will be displayed which have matching request type(s).

  • Scope Filter: You can select scope to filter the projects in Inbox.

10. By clicking on Save, System saves the selected checks on the respective sections.

Note: System will save the settings of the filter and show the same settings on the next login to the Inbox

11. By clicking on Apply Filters, system filters the projects on the basis of selected filters.

Note: By clicking on Cancel, filter pop up gets closed and navigates back to Reviewer Inbox

12. Search Projects: You can further filter the project(s) based on one or more criteria from the search section as displayed in the below screen

13. Search Criteria: You can select one of the search criteria from following dropdown values:

  • Project Reference Number

  • Project Title

  • Requesting Company

  • Third Party Company

  • Local Reference

  • Artist Name

  • Resource Title

  • Assigned To

  • Requestor Name

Note: By default, system displays the Project Reference No. as filter criteria

14. Click Search. System filters and displays the project on the basis of filters applied.

15. By clicking on Clear, system resets the filter criteria.

16. When both filter and search is applied, system shows the list of projects based on combination of filter and search criteria entered.

17. User Created Folders: By clicking on New Folder, the below pop up is displayed:

18. Enter the name of the new folder and click on Create.

Note : By clicking on Cancel, the New Folder pop up gets closed and navigate back to Reviewer Inbox.

19. System displays the new created folder in reviewer’s Inbox.

20. Edit User Created Folder: By clicking on Edit, the Edit folder pop up is displayed where you can edit the name of the folder.

21. By clicking Save, System saves the updated name in the Reviewer’s inbox.

Note: By clicking on Cancel, the New Folder pop up gets closed and navigate back to Reviewer’s Inbox.  

22. Delete User Created Folder: By clicking on Delete, system verifies if there are any projects existing in the folder which is getting deleted, if there are no projects in the folder, System asks for the confirmation with Delete and Cancel Button.

23. If there are projects in the folder, system displays a message “This folder contains project(s). To delete this folder, please move its project(s) to some other folder.

24. Move projects between folders: You can drag and drop the projects from the below listed folders in the inbox.

  • System folders to any of the user created folders

  • From any of the user created folders to any other user created folder

  • From any of the user created folders to its original system folder

Note: You cannot drag and drop the projects from/to artist consent folder.

Right Hand Side section of Reviewer’s Inbox

1. You can click on any column of the project except project reference number to view the right hand side section of the project.

2. Project Reference Number: RHS displays project reference number as hyperlink. You can click on Project Reference Number hyperlink to open complete project in a new browser/tab.

3. If you roll over the mouse on project reference number, the system will display the “project details” as added on project information tab by requestor.

4. Project Status: Project status is displayed in brackets next to project reference number hyperlink

5. Actions: The system displays the actions which can be performed on pending requests. The action buttons are enabled for the project depending on the role for which project is being reviewed.

6. The system displays the resource details under two sections:

  • Resources: The resources added on resource tab are displayed under resources section

  • Resources of Existing Releases: The tracks of existing releases which are a part of the project are displayed under this tab.

7. Assigned To: All the resource information is non editable by reviewer except assigned to field and review comments. The system displays all the users of the same workgroup in the assigned to drop down. You can select one of the users against each request and click on save icon.

8. Review Comments: This is applicable if the reviewer role is not UMGI marketing. The clearance system displays a review comments section against each resource pending for review. If the role is UMGI marketing reviewer, there is a common comments section available below the action hyperlinks.

9. The reviewer can enter the comments in the comments section and click on save icon.

10. The review comments entered by a reviewer before performing an action on the request should stay visible to all the reviewers in the same workgroup.

11. Once an action is performed by the reviewer, the review comments will be visible in the routing details for the relevant resource.

12. Review History: Review history is not applicable when the role of the reviewer is UMGI marketing reviewer or UMGI Global Clearance

13. The clearance system displays an icon at the end of each resource in the reviewer’s list to view review history

14. The reviewer can click on the review history icon to view the details regarding the previous review of the resource in clearance system.

15. The reviewer can click on export to excel link to view all the resource history details in an excel.

Note: System displays the review status as final review status of request in clearance system.

16. Re-Instated Requests: The clearance system shows an icon against each request if the request is re-instated in the project after cancellation.

Note: This is not applicable for UMGI marketing reviewer. The icon is only displayed in that reviewer workgroup’s inbox where the request has been re-instated. Once this is actioned by the reviewer, the icon is not displayed to the other reviewers.

17. UPCs allocated to New Releases: Next to resource and resources of existing release tabs, system displays the UPC allocated to the release(s) on the project if the release type is new.

 Note: The UPC(s) will be displayed as semicolon separated numbers

18. RCC Handler: RCC handler will display the name of the RCC handler linked to the project, if any.

19. Approve: Set the flag to one or more resource(s) and click Approve.  

20. Conditionally Approve:  Set the flag to one or more resource(s) and click Conditionally Approve.

Note: Adding comments is mandatory if a request is being conditionally approved. By clicking on Cancel, system discards the review action and displays the inbox screen.

21. Reject: Set the flag to one or more resource(s) and click Reject.

Note: Adding comment is mandatory when a request is being rejected.

22. If the comments are not added against the request being rejected, reject request popup is displayed.

Note: On clicking Cancel, pop up gets closed and navigates back to reviewer’s inbox.

23. You can select Predefined reason for rejection or can enter comments in comment section

24. Dispatch: Set the flag to one or more resource(s) and click Dispatch.

25. Dispatch pop up is displayed which displays the child/parent workgroup(s) associated with the reviewer’s workgroup to dispatch the request(s)

26. You can select one of the workgroups from the list and click on dispatch.

27. System will remove the request from the reviewer’s work group and route it to the dispatched workgroup.

28. Artist Consent : Set the flag to one or more resource(s) and click Artist Consent.

29. System creates the projects and respective request which is sent comes under artist consent folder.

30. By clicking on Undo icon against the project request to move the request back into its previous folder.

31. Route to RCC Admin Hyperlink: Set the flag to one or more resource(s) and click Route to RCC Admin. Adding comments is mandatory for this action to complete.

32.  If you have not added comments against the request, system displays the Route to RCC Admin pop up.

33. Enter the comments in the comments field and click on Route to RCC Admin button.

34. System will transfer the request in RCC Admin’s Inbox to take necessary action and request gets removed from the reviewer’s folder

Note:

1. System will update the status of the resource for Requested Resource and move the resource to next routing steps.

2. System will send the notification to the requester when request is fully reviewed.

3. The UMGI marketing reviewer cannot select a specific resource to perform the actions. Any action performed by UMGI marketing reviewer is applicable for the entire project

4. UMGI marketing reviewer can only perform approve, conditionally approve, reject and dispatch actions.

35. Refresh: By clicking on the Refresh icon, you can manually refresh the Inbox at any point in time.

36. Notifications: The Inbox will have a notification icon with a counter to show the number of unread notifications in the Inbox.

37. Click on specific notification, System will directly navigate the requestor to the project in the Inbox in a new browser tab for which the notification has been received and right hand side of the project will be displayed.

38. System also removes any other notifications for the same project for all the users in the Inbox. It also removes the bold highlight against the project in Inbox for the individual user.

39. If you click directly on the project in the Inbox without clicking on the project notification, System removes all the notifications related to the project from the Inbox of the workgroup.
