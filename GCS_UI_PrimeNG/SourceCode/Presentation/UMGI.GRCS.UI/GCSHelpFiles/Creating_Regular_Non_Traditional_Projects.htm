Creating Regular/Non Traditional Projects

*User with requestor role and having requestor rights can create a Regular/Non-traditional project in Clearance System

Fields marked with an asterisk * are mandatory fields

Project reference no. is only created after Save or Submit

User can skip between the tabs without saving

It is advised however, that users save regularly since there is not autosave function so in case of system issues data will be lost when not saved

1. Select Projects -> Create Projects -> Regular/Non Traditional Project. Click on Regular/Non Traditional Project Tab.

2. By default, system opens the project information tab with the following accordions:

  • Project Information

  • Third Party

Project Information

3. By default, System displays the following fields on the Project Information tab

  • Status of the project as UnSubmitted.

  • Created date  as today’s date

  • Requestor name

4. Project Title: You can enter the name of the Project Title.

5. Local Reference: You can enter text for Local Reference.

6. Details: You can enter the details related to project.

7. You can set the flag to select the following check boxes

  • Muti Artist

  • Compilation

  • High Priority

  • Sensitive Exploitation

  • One Stop

Note: If you set the flag one One stop check box, then it is mandatory to give one stop reason details in the text box.

8. Release Countries: To add territory/country to the project, click Manage Territories. The Manage Territory pop-up is displayed. For Description: Refer Manage Territories.

9. Requesting Company: By default, system displays the first company linked with the Requestor’s Workgroup.

10. If there is more than one company linked with the Requestor’s work group or if the Requestor is a part of more than one requestor workgroup, system displays the default company based on your settings in User Preferences.

11. You can select any other company which are linked with the Workgroup(s) from the requesting company drop down.

12. Estimated Sales/Units: Enter the Estimated Sales/Units up to 8 digits without decimal.

13. Third Party Repertoire: You can enter the details of Third Party repertoire as text in comments field.

14. Release Date: You can select the release date from calendar option.

15. Upload Project Documents: You can add related project documents by clicking on Browse button.

16. You cannot upload project documents which exceeds the size limit of 4MB, system displays the error message “Attached document exceed 4MB size limit”.

17. You can select multiple documents from same/ different folder one by one but total number of documents uploaded should not exceed 10. The maximum size of all the documents cannot exceed 4 MB limit.

18. Once selection of documents is done, you can upload the project documents by clicking on Upload button.

19. By clicking on Remove link, you can remove the uploaded documents.

20. Currency: You can select any other currency from the drop down the currency and can edit can be selected from any of the tabs.

21. Third Party: If the project is third party, then you can set the flag on third party and system will display the section to enter third party details.

Third Party Section

22. Search Third Party Company: To add the 3rd  party , you can search for the 3rd Party Company by entering one or more combination of the following search criteria and click Search.

  • By company Name

  • ISAC Code

  • Country

23. System can identify below search results based on the search criteria

  • Unique Search Result

  • Multiple Search Result

  • No search results

Unique Search Result

24. If system identifies a unique match for the company, then system adds the company and display details on the project information tab as displayed under:

Note: By clicking on Remove, system removes the added company.

Multiple Search Results

24. If system identifies more than one third party company, then system displays the search results in '3rd Party Company Search' pop up screen in alphabetical order.

25. You can set the radio button to select the third party company and click on Add.

26. After adding the 3rd party company, system navigates back to create new regular/non-traditional project  screen with the third party appearing on the screen with the following details.

Note: By clicking on Remove , system removes the added 3rd party company.

Add Free Hand Third Party Company

27. If system does not identify any match based on the search results, system opens the 3rd party company search pop up and enables Add freehand button shown below:

28. After clicking on 'Add Freehand' button, system displays the company name field to be entered.

29. By clicking Add, system displays the 3rd party company name with a flag to indicate free hand company and navigates back to the project information tab of create new regular/non-traditional project.

Note: By clicking on 'Remove' link , system removes the added free hand 3rd party company.

30. License Term: Enter the details in license term.

Request Type

31. Release Type: Navigating to the request type tab on the create project screen, system will enable the release type drop down. By default, system displays the release type as New.

Note: You can change the release type from drop down to existing.

32. Scope and Request Type: System displays the Scope and Request Type accordion with the following fields:

  • Scope

  • Request Type

33. Scope: You can set the flag to select Physical and/or Digital under scope section.

34. You can set the flag to select one or more request type from request type section.

35. If you select regular retail, you can either select for TV/Radio Break ICLA or Price reduction. You can choose to select either of these request types.

Note: *Price Reduction is only enabled when the release type is existing.
 

36. TV/Radio Break ICLA: System displays the following accordions when TV/Radio Break ICLA is selected.

37. Duration From and Duration To: Select the Duration from and Duration to from the calendar option.

38. Select at least one of the options from TV/ Radio/ Others.

39. System displays following options under each selected section of TV/Radio/Others.

  • Budget

  • Budget in USD: Budget in USD is mandatory if the currency selected at project level is not USD

  • Production cost of commercial

  • Other media details : Other media details is displayed only under Other section.

40. Sales Channel: Select one of the sales channels from the given below highlighted list

41. Campaign Countries: To add Campaign countries, click on Manage Countries. The Manage Territory pop-up is displayed. For Description: Refer Manage Territory in managing workgroup

42. Physical and Digital Sales Split: Physical Sales Split and digital sales split section contains following fields to be entered:

Note: Physical sales split section is displayed if physical scope is selected. Digital sales split section is displayed if digital scope is selected.

 

  • Sales to Date

  • Sales With

  • Sales Without

Note: You can enter 8 digits without decimal places in the above fields.

43. Physical Revenue Split and Digital Revenue Split: Physical Revenue Split and digital revenue split sections contains following fields to be entered:

Note: Physical revenue split section is displayed if physical scope is selected. Digital revenue split section is displayed if digital scope is selected.

  • Revenue to Date

  • Revenue With

  • Revenue Without

Note: You can enter 14 digits up to 2 decimal places in the above fields.

44. Total Sales Split: System displays the Total sales split section with following fields: this section is only displayed if both physical and digital scope is selected in the project

  • Sales to Date

  • Sales With

  • Sales Without

Note: You can enter 8 digits without decimal places in the above fields.

45. Digital Revenue Split contains following fields to be entered:

  • Revenue to Date

  • Revenue With

  • Revenue Without

Note: You can enter 14 digits upto 2 decimal places in the above fields.

46. If digital and physical options are selected in scope with request type as TV/Radio/Break ICLA, then system displays the following details to be filled for the following headers:

  • Sales to date

  • Sales with

  • Sales without

Note:

1. System sums up the “sales without” field for the physical sales split and digital sales split  sections and displays the results.

2. This field is non-editable.

47. System displays the Total Revenue split section with following fields:

  • Revenue to date

  • Revenue with

  • Revenue without

Note:

1. System sums up the “revenue without” field for the physical revenue split and digital revenue split  sections and displays the results.

2. This field is non-editable.

48. Price Reduction Section: If you have selected price reduction in the request type section, system displays the following details to be entered.

49. Current price level: By default, system displays current price level as top. You can change it to mid from dropdown.

50. Requested price level: You can select from dropdown.

51. Club Section: If you have selected club in the request type section, system displays the following details to be entered.

52. Non Traditional Section: If you have selected Non Traditional in the request type section, system displays the following details to be entered.

 

53. Client Name: Enter the text in Client name field.

Note: You can enter 100 characters including alphanumeric and special characters.

54. Client Website: You can enter the details in Client Website field.

55. Media Promo Spend Comments: You can enter the comments in Media Promo Spend Comment field.

56. Manufactured by UMG: Select Yes/No from Manufactured by UMG drop down.

57. Select one or more options by setting the flag against the following fields:

  • Partwork

  • Kiosk

  • Mail order

  • Internet

  • Direct Response

  • Educational

  • Premium

  • Give away free of charge

  • Other

58. Promotional Section: If you have selected Promotional in the request type section, system displays the Distribution To field to be entered.

Release Tab

*You can add existing/new release(s) under release tab depending on the release type selection.

Creating Regular/Non Traditional Projects with Existing Releases

1. Search Release: For existing release, after navigating to the release tab, you can search for an existing release in R2 by either one or more combination in below given fields and click on Search.

Note: if unique result is returned by the system, it should get added on the release tab itself. If no results are identified, system will highlighted it on the release tab itself.

2. The relevant search results for more than one search result are displayed with the following details in Search for Release Pop up.

3. You are not allowed to add duplicate releases on the project.

4. To view the track details, click on the arrow after the select option for each release.

5. To add the existing release(s) to the project, click on Add to Project.

6. After adding release, system displays the below screen.

7. By clicking on Remove link, system removes the added release

Note: System disables the remove link against the release after the project is submitted.

8. Comments: You can enter the comments for existing release(s) in the project.

9. View Tracks: You can expand the tracks section to view tracks of the added release.

Creating Regular/Non-Traditional with New Releases

1. To add new release(s) to the project, navigate to the Release tab and you can see the below screen with the fields to enter.

2. Add Artist on New Release: To add Artist to the new release, click on Manage Artist and system displays the pop up to search for an artist.

3. You can search Artist by Artist Name and Artist ID and click Search.

Note: By clicking on Reset, system clears the entered search criteria.

4. System displays the search results with the following fields:

5. User select the relevant Artist and clicks on ADD, System navigates back to the release tab.

Note: If you want to add more than one artist in a new release, click on Manage artist again.

6. If you wish to remove the added artist, click on manage artist button again and select the artist to be removed.

7. Click Remove Button, System can remove the already linked artist(s).

8. Release Title: Enter the Title for the new release as text.

Note: For the first release on the project, the project title is displayed as the release title by default.

9. Version Title: You can enter the details of version title as text.

10. Label: Enter the label in Label field.

11. Configuration Group: Select the configuration group from the drop down.

12. Configuration: Based on the configuration group selected, select one of the configuration options from the drop down.

13. Music Type: You can select Music Type as one of the drop down values.

14. Sound Track: You can set the flag to select sound track for the new release

15. Number of Tracks: You can enter the number of tracks for a new release.

16. Package: You can the set the package for a new release as Yes/No.

17. Number of Components: If Package is set to yes, then add the number of components field to complete the package.

18. Existing Releases in Package: To add existing release(s) in the package for the newly created release then select yes from the dropdown of existing releases in package and click on Search.

19. Search for Release pop up is displayed with the search criteria to be entered. For Description: Please Refer Search for Release Pop Up

20. View/Remove Existing Release: Click view existing releases to view the existing releases added in the package.

21. View Components: Select one of the UPCs to view the details on the RHS of the screen.

Note: By Clicking on Remove Link to remove the added component against the release details in the view components screen.

22. If package is selected as No, system does not display “no of components” and “existing releases”.

23. Comments: You can add comments against the release.

24. In case you want to add another new release on the project, click on “Add Release” button. system will display a blank template to add the release.

25. Duplicate Release: You can also add a new release by duplicating the existing release.

Note: Duplicate button will be enabled once all the mandatory fields of a release are entered.

26. System displays the copied release and you can edit the relevant fields in the duplicated releases.

27. If you wish to duplicate a release by changing the configuration, select one of the configurations shown in the dropdown.

28. Click on Duplicate button. System copies all the fields and changes the configuration to the selected one.

Effect of selection of Request Type on Release Tab

1. Regular Retail Section: When Regular retail is selected in the request type tab, system displays the regular retail section on the release tab with the following fields:

Note: If TV/break and/or price reduction is selected at request type tab, then regular retail section will not appear against the release.

2. Price Level: By default, system displays the ‘Top’ as the price level. You can set the value from the drop down.

3. ICLA Level: By default, system displays the ICLA level values which is calculated by the system and is non- editable. You can select the deviated ICLA level flag if applicable. The value of deviated ICLA level, can then, be selected from dropdown.

4. Comments: Enter the ICLA Deviation comments as text in comments field.

5. TV/Radio Break ICLA Section: System shows the TV/Break ICLA accordion with the following field, when the flag on TV/Break ICLA is set on Request type tab.

6. Price Level: By default, system displays the ‘Top’ as the price level. You can set the value from the drop down

7. ICLA Level: By default system displays the ICLA Level as TV/Break ICLA.

Note: * It is a non-editable field.

8. Deviated ICLA Level: You can select deviated ICLA level. If that is selected, then enter the ICLA Deviation comments as text in comments field.

9. Exact PPD: Enter the details as currency for exact PPD field..

10. Estimated Retail: Enter the details as currency for Estimated Retail.

11. Price Reduction Section: When the price reduction is set as yes in the request type, then system displays the price reduction accordion with the following fields:

12. Price Level: By default system displays the Price level as Requested Price level.

Note: * It is a non-editable field.

13. ICLA Level: ICLA level is displayed by system and is non-editable by the user.

14. If you set the flag on deviated ICLA level, system enables the drop down and you can choose the drop down values.

15. Comments: Enter the ICLA Deviation comments as text in comments field if deviated ICLA level flag is checked.

16. Club Section: System displays the club section on Release tab, when you set the flag to select club on request type tab.

17. Price Level: By default, system displays the price level as club

18. ICLA Level: By default, system displays the ICLA level as Club.

Note: * It is a non-editable field.

19. Deviated ICLA Level: If you set the flag on deviated ICLA level, system enables the drop down and you can choose the drop down values.

20. Comments: Enter the ICLA Deviation comments as text in comments field if deviated ICLA level is selected.

21. Promotional Section: System displays the promotional section on release tab with the below given fields, when you set the flag to select on promotional on request type tab.

22. Price Level: System displays the price level as NIL which is non-editable.

23. ICLA Level: System displays the ICLA level as NIL which is non-editable.

24. Deviated ICLA Level: If you set the flag on deviated ICLA level, system enables the drop down and you can choose the drop down values.

25. Comments: Enter the ICLA Deviation comments as text in comments field if deviated ICLA level is selected.

26. Non Traditional Section: System displays the non-traditional section on release tab with the below given fields, when you set the flag to select on Non Traditional on request type tab.

27. Select one of the options from ICLA or suggested fee. Selecting one is mandatory.

28. ICLA Level: If you select ICLA, system displays the ICLA level by default which is non-editable.

29. Deviated ICLA Level: If you set the flag on deviated ICLA level, system enables the drop down and you can choose the drop down values.

30. Comments: Enter the ICLA Deviation comments as text in comments field, if deviated ICLA level is selected.

31. Invoice Price: Enter the currency on invoice price field on release screen.

Note: This field is only enabled when manufactured by UMG is set to yes on Request Type Screen.

38. Selling Price Less VAT: Enter the details as currency in Selling Price less VAT field

Note: This field is enabled only when manufactured by UMG is set to NO and Give away of free recharge is not selected.

39. ICLA Accounting Base: Enter the currency for the ICLA accounting base field.

Note:

1. This field is enabled when manufactured by UMG is set to Yes and partwork is selected on Request type screen.

2. This field is enabled when manufactured to UMG is set to NO and has not selected give away free of charge.

40. Deemed PPD: Enter the currency for the Deemed PPD field.

Note: This field is enabled when manufactured by UMG is set to NO and give away free of charge is selected on request type screen.

41. Resource Fee: Enter the currency for the Resource fee field.

Note: This option is available when suggested fee is selected in the radio button
 

Resource Tab

42. When the Release Type is New, system enables resource tab to enter resources on the project. For Description: please refer the Resource tab.

Note: You can add resource(s) which are part of existing release(s).

43. Save: You can click on Save button to save the project at any point in time. When Project is successfully saved, system will populate the project reference number on the project information tab.

44. Submit: Click on Submit, once all the mandatory fields are entered and system opens the request summary tab to display the current status of all the requests.

Note: Summary tab should be manually refreshed by click on Refresh on Request summary tab whenever you open the project.

Request Summary

1. You are navigated to request summary tab, once the project is successfully submitted to display the current status of all the requests. System will create one request for each request type and sales channel added on the project for every resource.

2. System displays two sections on the summary tab:

  • The first section displays the list of all the request types for resources added on resource tab.

  • The Second section “resources of existing release” displays all the requests for tracks of existing releases added on the project.

3. System displays the following non editable fields on the screen.

  • UPC

Note: Only applicable for resources of existing release.

  • Resource Title :- Displayed as Artist: Resource title, Version title

  • ISRC

  • Resource Type

  • Clearance Owner

  • Request Type

  • Configuration

Note: Only applicable for resources of existing release.

  • Submission Date

  • Reviewer Role

  • Reviewer Workgroup

  • Review status

  • Review Date

  • Comment

  • Routing Details
