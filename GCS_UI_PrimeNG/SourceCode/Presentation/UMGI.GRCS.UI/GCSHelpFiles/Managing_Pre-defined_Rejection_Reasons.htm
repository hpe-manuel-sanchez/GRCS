Managing Pre-defined Rejection Reasons

* The action on Manage Pre-defined Rejection Reasons can only be performed by RCC Admin

1. Login to GCS and the home page of the system is displayed.

2. Select Admin ->Manage Pre-defined Rejection Reasons. Click on Manage Pre-defined Rejection Reasons button.

3. Manage Pre-defined Rejection Reasons screen  is displayed  which displays the list of reasons already created in the system:

4. Create a New Pre-Defined Rejection Reason: The RCC Admin can write free text in the predefined reason for rejection text field.

5. System displays the “applicable to” column with Marketing, Legal and UMGI options selected by default. You can select/de-select one or more options.

6. Enter the relevant Pre-defined Rejection Reasons and select at least one option in Applicable to field.

Notes:

1.  The rejection reasons where marketing is selected in the applicable to field will be visible in the pre-defined rejection reasons pop up to all the national marketing reviewers, international marketing reviewers and local label reviewer workgroup user when they reject a request

2. The rejection reasons where legal is selected in the applicable to field will be visible in the pre-defined rejection reasons pop up to all the national and international legal reviewer workgroup users when they reject a request for legal review

3. The rejection reasons where UMGI is selected in the applicable to field will be visible in the pre-defined rejection reasons pop up to all the UMGI marketing reviewers and UMGI global clearance reviewer workgroup users when they reject a request

4. On clicking Add, the entered pre-defined rejection reason is added in the list of existing reasons and again blank text field is displayed. The latest reason added appears on the top of the list of existing reasons.

5. Edit Pre-Defined Rejection Reason: On clicking Edit icon, s.no, text area and applicable to check boxes gets enabled to be edited.

6. Edit the text: You can change the language of the text written for rejection reason

7. Edit Serial Number: You can change the s.no of the added reason by changing the number in the text box. You can enter any number upto maximum of 3 digits.

Note: System displays the pre-defined rejection reasons to the reviewer based on the sequence assigned to these on the manage predefined rejection reasons screen. In case a particular s.no. is not applicable for a role, the system will ignore the same and create the sequencing in the dropdown in pre-defined rejection reason pop up  based on available S.No.s for that role.

8. Editing Applicable to Option: You can set the flag to select any other options or de-select the already selected option.

Note: Selection of at least one Applicable to option is mandatory.

9. Undo: On Clicking Undo, system reverts back to the previous state without saving any changes. This is applicable before clicking on save.

10. System discards all the changes and makes the text, S.No and applicable to fields non-editable with delete and edit icons displayed again.

11. Save: You can make the changes and click on Save.

12. System updates the details based on the saved changes and makes it non-editable with delete and edit icons displayed again

13. Delete: On clicking Delete icon, system deletes the selected pre-defined rejection reason.
