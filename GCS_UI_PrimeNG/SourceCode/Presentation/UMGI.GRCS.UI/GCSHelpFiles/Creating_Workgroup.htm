Creating Parent Workgroup

*Only RCC Admin can create parent workgroup in the Clearance System.

Fields marked with an asterisk * are mandatory fields

1. Login to GCS and the home page of the system is displayed.

2. Select Workgroup -> Create Workgroup. Click the Create Workgroup tab.        

3. The Create New Workgroup screen is displayed with default status as WIP.

4. Workgroup Name: Enter the name of the workgroup as text.

5. Workgroup Role: Select a role from the following Role drop-down values.

  • Local Label Reviewer

  • National Marketing Reviewer

  • National Legal Reviewer

  • International Marketing Reviewer

  • International Legal Reviewer

  • UMGI Marketing Reviewer

  • UMGI Global Clearance

  • Requestor

  • Inquiry

Manage Users

6. Manage Users: To add users to workgroup, click the Manage Users button. The Manage Users of Workgroup pop-up is displayed. The pop-up is empty if users for workgroup are to be added for the first time.

Note: Before Managing Users, selection of role is mandatory.

7. You can search users based on the following search criteria:

  • Search by User Name

  • Search by User ID

  • Search by Country

8. You can enter text in the search criteria fields and click Search. The system displays a list of users based on the above search criteria in the pop-up in an alphabetical order.

Note:  If you click Reset, all the search criteria entered will be cleared, ready for entering new search criteria.

9. Select the check box corresponding to the user you wish to choose and click Add. You can select one or more users.

10. The pop-up displays only selected users in the list with the user details such as contact email ID and the Rights.

Note:  If you click Remove, the selected users are removed from the list.

11. Default User: You can set one or more user as default and assign rights to each of the selected user.

Notes:

1. The default flag will only be enabled if role of the workgroup is reviewer.

2. By default, system sets the flag on the rights type based on the role of the work group.

3. You can select/deselect any of the rights against the user.

12. Click Save. The pop-up is closed and the selected users are displayed in an alphabetical order in the Create New Workgroup screen.

Note: If the number of users selected is only one, the system automatically selects the default flag for the user on Save.

Manage Company

Notes:

1. Adding a company is mandatory for all the roles except inquiry.

2. Inquiry can have either company or territory/country.

3. For UMGI marketing reviewer and UMGI global clearance reviewer roles, a company cannot be added.

13. To add company to the workgroup, click the Manage Company button. The Manage Company pop-up is displayed. The pop-up will be empty if companies for the workgroup are to be added for first time.

14. You can search company based on the following search criteria:

  • Search by Company Name

  • Search by ISAC Code

  • Search by Country

15. You can enter text in the search criteria fields and click Search. The system displays a list of companies based on the above search criteria in the pop-up and are sorted by country first and then by company alphabetically.

16. Select the check boxes corresponding to the company you wish to choose and click Add. You can select one or more company.  

17. The Manage company pop-up displays only selected companies in the list.

18. Review the list of company(s) added to the workgroup and click Save. The pop-up is closed and the selected companies are displayed in the Create New Workgroup screen sorted alphabetically by country and then company.

Note: If you click Remove, the selected company(s) are removed from the list.

Manage Territories

19. Manage Territories: To add territory/country, click Manage Territories. The Manage Territory pop-up is displayed.

Notes:

1. Manage territories is only displayed if role is local label or inquiry.

2. It is mandatory for local label but optional for inquiry.

20. In this pop-up, the countries are categorized and displayed hierarchically. For examples Universe -> World -> Territory -> Countries. By default, the countries and territories are displayed in a collapsed view under each territory grouping. Each section can be expanded by clicking the arrow alongside the header.

21. The Manage Territories pop-up displays three columns: Include, Exclude and N/A. By default, the list of countries and territories are all selected as N/A.

22. You can search for a specific country or territory in the Auto Suggest search box. Enter the name of the Country/Territory required and click Find. The country or territory that matches the search criteria is displayed along with its hierarchy. For example, if you search for the country ‘Algeria’, then the hierarchy will be expanded under Universe -> World->Africa->Algeria.

Notes:

1. If you select a country which is present in two territories, then the country is displayed in a row under each related territory.

2. If you click the Clear Filter hyper link, then the searched country/territory is removed and the original collapsed view is displayed.

23. To add country/territory, select the Include option button against the country/territory. The selected country/territory is then displayed in the Include list text box on the RHS.

24. To remove country/territory, select the Exclude option button. The selected country/territory is then displayed in the Exclude text box.

Notes:

1. Whenever a higher level in hierarchy of territories/countries is ‘selected’ in the list, all lower levels will be auto selected – and displayed in alphabetical order.

2. The above point is valid, if one of the countries out of a selected territory is to be removed.

3. If the complete territory is to be removed, then exclude is not allowed

25. Once your selections are complete, click Save. The Manage Territory pop-up is then closed and you are returned to the Create New Workgroup screen. The selected country/territory is displayed in the Manage Territory field.

26. You can view the countries in each territory by scrolling the mouse over the territory.

27. Enter the R2 contact email ID in the R2 Contact field.

28. Click Create. The workgroup is created and the system automatically changes the status of the work group as ‘Active’.

29. System creates the parent workgroup based on the following fields uniqueness check:

  • Role

  • Company

  • Country (Include) (In case of ‘Inquiry’ and ‘Local label Reviewer’ workgroup)

Note:  Parent uniqueness is validated against the other parent workgroups with the above mentioned fields and not the child workgroup.
