Requestor Inbox

1. Login to GCS and the home page of the system is displayed.

Note: Home page of GCS system is Inbox view.

2. If you are a part of Reviewer as well as Requestor role, System displays 2 tabs on the home page, for Reviewer and for Requestor. You can view the default Inbox based on the settings done on User Preferences screen.

3. Similarly, if you are a part of RCC Admin role, system displays RCC Admin Inbox as Default. Click on Requestor tab to view and perform requestor Inbox actions.

4. Inbox is divided into two sections: Left hand side and Right hand side. You can drag the divider between two sections to increase/decrease the view of each section.

Left Hand Side section of Requestor’s Inbox

1. Grouping of Projects: System groups the projects in the following folders for Requestor’s Inbox:

  • High Priority

  • Submitted

  • Un Submitted

  • Re-Opened

2. Default Folder : Whenever you log in to the Global Clearance System, by default, System displays the High Priority folder as expanded mode. If the high priority folder is empty, the system opens the submitted folder by default.

Note: Only one folder can be in expanded mode at a time. If the user clicks on any other folder, the previously opened folder will become collapsed.

3. By default, System displays 20 requests in each folder. If the folder contains more than 20 requests, the system enables the Show All link. You can click on Show All link to view all the projects within the folder.

Note: System changes the Show All link to Show Default link to revert to the default mode.

4. All the projects in each folder will appear in bold when a notification has been received for the request. Once the requestor has clicked on a bold project, the bold highlight will be removed.

Note: The requestor can click on the “eye” icon to again highlight the project as bold to indicate unread.

5. System shows an indicator against the project for which review of all the requests is completed.

6. System shows the following columns in the sequence listed below for each project.

  • Project Type

  • Project Reference Number

  • Project Title

  • Submission Date

  • Notification Date

  • Requestor Name

  • Requesting Company

  • Third Party Company

  • Release Date

7. Click on Project Reference Number hyperlink, project opens in the new browser tab.

Note: By default, Project information tab is opened in editable mode.

8. Filters: By clicking on Filters hyperlink, system opens the filters popup with the below given sections:

  • Request Type Filter: You can select one or more request types to filter the projects in inbox. Only those projects will be displayed which have matching request type(s).

Note:

1. System will not filter the projects based on request type in UnSubmitted folder.

2. All the projects of UnSubmitted folder will continue to be displayed.

  • Requestor Filter: System displays the list of users who are a part of the same requestor Workgroup(s) with which you are linked.

Note: More than one requestor can be selected from the dropdown to filter the projects in inbox.

  • Scope Filter: You can select scope to filter the projects in inbox.

Note:  The clearance system will not filter the projects based on scope in UnSubmitted folder. All the projects of UnSubmitted folder will continue to be displayed.

9. By clicking on Save, System saves the selected checks on the respective sections of the filter.

Note:  System will save the setting of the filter and show the same settings on the next login to the inbox.

10. By clicking on Apply Filters, system filters the projects on the basis of selected filters. These setting will not be stored during next login.

Note: By clicking on Cancel, filter pop up gets closed and navigates back to Requestor Inbox.

11. Search Projects : You can further filter the project(s) based on one or more criteria from the search section as displayed in the below screen.

12. Search Criteria: You can select one of the search criteria from following dropdown values:

  • Project Reference Number

  • Project Title

  • Requesting Company

  • Third Party Company

  • Local Reference

  • Artist Name

  • Resource Title

Note: By default, system displays the Project Reference No. as search criteria.

13. Click Search. System filters and displays the project on the basis of search applied.

14. By clicking on Clear, system resets the search criteria.

15. When both filter and search is applied, system shows the list of projects based on combination of filter and search criteria entered.

16. You can click on any column of the project except project reference number to view the right hand side section of the project.

Right Hand Side section of Requestor’s Inbox

1. Project Reference Number: RHS displays project reference number as hyperlink. You can click on Project Reference Number hyperlink to open complete project in a new browser/tab.

2. Project Status : Project status is displayed in brackets next to project reference number hyperlink

3. Project Actions: The following project actions are displayed on mouse roll over based on the status of the project:

  • Mass Reminder

  • Mark as Completed

  • Mark as Cancelled

Note: Project actions link will not be displayed for right hand side for an UnSubmitted project is opened.

4. Mass Reminder: Click on Mass reminder, system will open a mass reminders pop up.

5. System displays all the requests with waiting status with resource details, request type and ISRC. You can set the flag to select all or one or more requests and click Send Reminder.

6. System will send the reminder notification for all the selected pending requests to the relevant reviewers.

7. Mark as Completed: You can click on mark as completed to complete the project from inbox.

8. The status of the project will be changed to completed and all the fields will become non-editable. Refer Project Completion.

Note: On manual refresh of inbox, the project will be removed from the left hand side section and can be accessed by navigating to manage cancelled/ completed project link.

9. Mark as Cancelled: You can click on mark as cancelled to cancel the project from inbox.

10. The status of the project will be changed to cancelled and all the fields will become non editable. Refer Project Cancellation.

Note: On manual refresh of inbox, the project will be removed from the left hand side section and can be accessed by navigating to manage cancelled/completed project link.

11. Display of Resources: System displays the resources under two tabs:

  • Resources Tab

  • Resources of Existing Release Tab

Note: The above tabs are not applicable for projects opened from UnSubmitted folder.

12. UPCs allocated to New Releases: Next to these two tabs, system displays the UPC allocated to the release(s) on the project if the release type is new.

Note: The UPC(s) will be displayed as semicolon separated numbers.

13. RCC Handler: RCC handler will display the name of the RCC handler linked to the project, if any.

14. System displays an indicator against each of the request for which routing has been completed.

Note: System displays this indicator against each resource line only when all the requests for the resource have been reviewed (irrespective of the review status).

15. By Clicking on UnSubmitted project projects type , system displays the right hand section with following details under two tabs:

  • Resources: Displays the list of resources added on resource tab

  • Resources of Existing Releases: This section displays the release information for UnSubmitted project.

16. Inbox Refresh: By clicking on the Refresh icon, you can manually refresh the Inbox at any point in time.

17. Notifications: The Inbox will have a notification icon with a counter to show the number of unread notifications in the Inbox.

18. Click on specific notification, System will directly navigate the requestor to the project in the Inbox in a new browser tab for which the notification has been received and right hand side of the project will be displayed.

19. System also removes any other notifications for the same project for all the users in the Inbox. It also removes the bold highlight against the project in Inbox for the individual user.

20. If you click directly on the project in the Inbox without clicking on the project notification, System removes all the notifications related to the project from the Inbox of the workgroup.
