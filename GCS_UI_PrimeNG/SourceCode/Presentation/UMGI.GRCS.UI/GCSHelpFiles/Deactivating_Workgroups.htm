Deactivating Workgroups

*RCC Admin Workgroup can deactivate a parent or child workgroup.

*Workgroup Users who have manage workgroup right can deactivate only a child workgroup.

1. Select Workgroup -> Deactivate Workgroup. Click on Deactivate workgroup tab.

2. The Deactivate Workgroup screen is displayed.

3. You can search the workgroup through the available search by options, based on either one or combination of the search criteria:

  • Search by workgroup name

  • Search by role in drop-down

  • Search by company name and/or ISAC Code

  • Search by user name

  • Search by Territory/Country  name

4. Enter the search criteria and click Search. The active workgroups that match the search criteria are displayed as list in an alphabetical order.

5. Select a workgroup you wish to deactivate by selecting the option button corresponding to that workgroup.

6. If you wish to view the details of workgroup before deactivating it, click View. The View Workgroup pop-up is displayed with workgroup details.

7. You can view the details and click Deactivate to deactivate the workgroup or Cancel to cancel.

8. If you wish to deactivate the workgroup without viewing its details, click Deactivate in the Deactivate Workgroup screen.

9. If the selected workgroup is Parent workgroup and if there are no pending requests w.r.t the parent and its children workgroup, then

  1. The system deactivates the parent and associated child workgroups.

  2. The parent and child workgroups disappear in the list on the deactivate workgroup screen.

  3. The system updates the status of the parent and child workgroups as inactive.

Note:  If the selected workgroup is Child workgroup with no pending requests, then system deactivates and updates the status of the child workgroup as inactive with no impact on parent workgroup.

10. If the selected workgroup is Parent workgroup and if there are pending requests associated with the parent or associated child for the workgroup, then

  1. A pop-up listing pending requests for both parent and associated child workgroups are displayed.

  1. Select the check box corresponding to the requests and click Select Workgroup. The Search and Redirect screen is displayed.

  1. Enter text as search criteria for the following fields to search for workgroups

  2. Click Search. The system displays the list of workgroups matching the search criteria and having the same role as the workgroup that is getting deactivated. The search results are displayed in alphabetical order.

  3. Select the check box corresponding to the workgroup to select the workgroup to which the pending requests should be routed and click Redirect. The system redirects all the pending requests to the selected workgroup.

  4. The system updates the status of the parent and child workgroups as inactive.

11. If the selected workgroup is Child workgroup and if there are pending requests for the workgroup, then

   a. A pop-up listing pending requests for the only child workgroup is displayed.

   b. Select the check box corresponding to the requests and click Select Workgroup. The Search and Redirect screen is displayed.

  c. System automatically displays the parent workgroup and associated child workgroups to redirect the pending requests in the search screen.

  d. Sets the flag to select the workgroup to which the pending requests should be routed and click on Redirect.

  e. System removes the selected requests from the list of pending requests.

Note: System updates the status of the workgroup as inactive when all the pending requests are redirected.
