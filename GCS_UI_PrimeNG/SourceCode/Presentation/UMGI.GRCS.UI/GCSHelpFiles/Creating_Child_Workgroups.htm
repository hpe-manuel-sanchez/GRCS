Creating Child Workgroups

* RCC Admin or workgroup users with manage workgroup rights can create the child workgroup(s) for the parent workgroup.

Fields marked with asterisk (*) are mandatory.

1. Select Workgroup -> Manage Workgroup. The Maintain Workgroup screen with search options is displayed.

2. You can search the workgroup through the available search by options, based on either one or combination of the search criteria:

  • Search by workgroup name

  • Search by role in drop-down

  • Search by company name and/or ID

  • Search by user name

  • Search by Territory/Country  name and/or ID

3. Enter the search criteria and click Search. The workgroups that match the search criteria are displayed as list in an alphabetical order.

4. Click on the workgroup name (hyperlink) from the list to create child workgroup and you are navigated to the parent workgroup screen with the parent details.

5. Click the Create Child Workgroup button. The Create Child Workgroup screen is displayed with blank fields blank except of the role, countries (if applicable) and R2 Contact.

Note: Create child workgroup button appears on the screen only when the role of the workgroup is reviewer.

6. Workgroup Name: Enter the name of child workgroup as text.

7. Manage Company: Click Manage Company. The Manage Company pop-up is displayed with the list of parent workgroup companies

Note: Only those companies can be added to child workgroup which are a part of parent workgroup

8. Search the list using the search criteria available and select one or more companies by selecting the check box corresponding to companies and click Add.

9. The selected companies are only displayed in the list.

10. Click Save. You are navigated to the Create Child Workgroup screen and it displays the above added companies under the company field in the child workgroup.

Notes:

1. If you click Remove, the selected company(s) is removed from the list.

2. If no company is selected for child workgroup, then the companies of parent workgroup is applied for child workgroup, but are not displayed in the screen.

11. Click Manage Users. The Mange Users for Workgroup pop-up is displayed. For Description: Refer Manage_Users section of Creating Parent Workgroup.

Manage Request Type

12. Request Type : You can select one or more request type by selecting the check box corresponding to the request type available.

Manage Artist Contract

13. Manage Artist Contract:  Click Manage Artist Contract. The Search for Artist Contract pop-up is displayed. The pop-up will be displayed if new child workgroup is created.

14. You can search artist contract based on the following search criteria:

  • Search by Artist Name

  • Search Artist ID

  • Search by Clearance Admin Company

15. You can enter search criteria fields and click Search. The system displays a list of artist contracts in the pop-up and is sorted alphabetically by artist name.

16. System displays the contract flag icon colour as “Green” if contract is linked with the Artist in GRS.

18. System displays the contract flag icon as “Red” if no contract is linked with the Artist in GRS.

19. You can click the arrow icon against the Artist to view the contract details.

20. If multiple contracts is available for the above selected Artist in GRS, the system displays the list of contract(s) of the Artist.

21. Select the check box corresponding to the contract to select it or click the ‘Select All’ to select all the contract in the pop-up.

22. Click Add. You are navigated to the Manage Artist Contract screen with the recently added contracts displayed at top of the page sorted alphabetically by artist name and clearance Admin company.

23. Click Save. You are navigated back to the create child workgroup screen with saved contract (s) sorted alphabetically by Artist Name and Clearance Admin Company.

Notes:

1. The system only links those contracts where clearance Admin company of the contract is a part of the workgroup.
2. The contract ID is displayed as a hyper link on the child workgroup screen and you click the hyper link to view its details.

Manage Resource Contract

23. Manage Resource Contract: Click the Manage Resource Contract. The Search for Resource Contract pop-up is displayed.

24. You can search the resource contract based on one or combination of following search criteria fields:

  • Artist Name

  • Artist ID

  • Title

  • Version Title

  • ISRC

  • Request Type

  • Clearance Admin Company

25. You can enter text as search criteria and click Search. The system displays a list of resource contracts in the pop-up and is sorted alphabetically by artist name.

26. System displays the contract flag icon colour as "Green" if contract is linked with the resource in GRS.

27. System displays the contract flag icon as "Red" if no contract is linked with the resource in GRS.

28. You can click the arrow icon against the resource to view the contract details.

29. If the contract is available for the above selected resource in GRS, the system displays the list of contract (s) of the resource as shown below:

30. Select the check box corresponding to the contract to select it or click the ‘Select All’ to select all the contract in the pop-up.

31. Click Add. You are navigated to the Manage Artist Contract screen with the recently added contracts displayed at top of the page sorted alphabetically by artist name and clearance Admin company.

Notes:

1. System links only those contracts where Clearance Admin Company of the contract is part of the work group.

2. By clicking on Remove Button, system can remove the added contract.

32. Click Save. You are navigated back to the create child workgroup screen with saved contract(s) sorted alphabetically by Artist Name, resource title, and Clearance Admin Company.

Note: The contract ID is displayed as a hyperlink on the child workgroup screen and you click the hyperlink to view its details.

33. Click Create. The child workgroup is created with status as “Active”.
