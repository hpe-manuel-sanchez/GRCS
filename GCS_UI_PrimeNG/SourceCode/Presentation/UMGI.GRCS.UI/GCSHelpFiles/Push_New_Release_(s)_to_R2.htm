Push New Release (s) to R2

Fields marked with an asterisk * are mandatory fields

1. Login to GCS and the home page of the system is displayed.

2. Navigate to the Requestor Inbox and click on the Project Reference Number to add release(s) to the project.

3. System enables the Push to R2 button.

Note: If the push to R2 is happening first time on the project, only R2 authorized user is allotted to do first push otherwise system displays error message. However, subsequent pushes can be done by anyone having requestor role.

4. Click Push to R2 button.

Note: If the push to R2 is happening first time on the project, only R2 authorized user is allotted to do first push otherwise system displays error message. However, subsequent pushes can be done by anyone having requestor role.

5. The pop-up is displayed with following options

Note: This pop up will only display if it’s a first push and the user is R2 authorized.

6. After selecting one of the above options, click on Submit button.

Search Existing Project in R2

7. Search for R2 project screen is displayed.

8. Enter one or more combination as an input to the search criteria.

Note:  Entering company as one of the search criteria is mandatory.

9. Click Search. System finds one or more R2 projects matching the search criteria.

10. You can select the relevant project and click on Link to Project.

Create New R2 Project

11. Enter the following fields to create R2 project.

12. Click Link to Project.

13. System displays the R2 Project ID on the screen and pushes the releases and resources.

14. Push to R2 cannot be executed for the following cases:

  • If project has a  Free Hand Third Party Company.

  • Resources having cancelled, rejected and freehand resources.

  • Releases without a UPC will not be pushed.

15. On subsequent pushes, only the additional releases and resources where were not pushed earlier will be pushed.

16. In case push for any of the resource or release is unsuccessful, system will send an email to the user who has pushed the release/resource after processing is completed in R2 (approx. in 1 hour)
