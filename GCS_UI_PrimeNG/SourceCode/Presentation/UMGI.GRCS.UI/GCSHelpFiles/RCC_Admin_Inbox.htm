RCC Admin Inbox

* Only RCC Admin can perform the actions on RCC Admin tab

* Only Submitted project will be displayed in the Inbox of RCC Admin

1. Login to GCS and the home page of the system is displayed.

2. By default, system displays the RCC Admin Inbox with all the submitted/resubmitted projects based on routing.

3. Inbox is divided into two sections: Left hand side and Right hand side. You can drag the divider between two sections to increase/decrease the view of each section.

Left Hand Side section of RCC Admin’s Inbox

1. Grouping of Projects: System groups the project in the following folders for RCC Admin Inbox

  • High Priority : This folder includes all the projects where the requests have been identified as orphan and have high priority flag set on the project except one stop routing requests

  • Orphan : This folder includes all the projects where the request(s) have been identified as orphan and does not have a high priority flag set

  • One Stop Routing: This folder includes all the projects where one stop flag is enabled

  • User created folder

2. Default Folder: Whenever you log in to the Global Clearance System, by default, System displays the Orphan folder as expanded mode. If the Orphan folder is empty, the system opens the High Priority folder by default.

Note: Only one folder can be in expanded mode at a time. If the user clicks on any other folder, the previously opened folder will become collapsed.

3. By default, System displays 20 requests in each folder. If the folder contains more than 20 requests, the system enables the Show All link. You can click on Show All link to view all the projects within the folder.

Note: The clearance system changes the Show All link to Show Default link to revert to the default mode.

4. All the projects in each folder will appear in bold when a notification has been received for the request. Once the requestor has clicked on a bold project, the bold highlight will be removed.

Note: The RCC Admin can click on the “eye” icon to again highlight the project as bold to indicate unread.

5. System shows the following columns in the sequence listed below for each project.

  • Project Type

  • Project Reference Number

  • Project Title

  • Notification Date

  • Submission Date

  • Release Date

  • Estimated Sales

  • RCC handler

6. Click on Project Reference Number hyperlink, project opens in the new browser tab.

7. Manual Allocation of RCC Handler: By default, Release tab is opened .You can select the RCC Handler from the dropdown and click Save.

7. Filters: By clicking on Filters hyperlink, system opens the filters popup with the below given sections

  • Request Type Filter: You can select one or more request types to filter the projects in Inbox. Only those projects will be displayed which have matching request type(s).

  • No RCC Handler Assigned: Set the flag to select only those projects which are not assigned to RCC Handler.

  • RCC Handler: Select the name of one or more RCC handlers from the dropdown to filter the projects based on RCC handler.

Note: The dropdown will only show the names of users who are a part of RCC Admin workgroup.

  • Scope Filter: You can select scope to filter the projects in Inbox.

9. By clicking on Save, System saves the selected checks on the respective sections of the filter

Note: System will save the setting of the filter and show the same settings on the next login to the Inbox.

10. By clicking on Apply Filters, system filters the projects on the basis of selected filters.

Note: By clicking on Cancel, filter pop up gets closed and navigates back to RCC Admin Inbox.

11. Search Projects: You can select one of the search criteria from following dropdown values:

  • Project Reference Number

  • Project Title

  • Local Reference

  • Requestor Name

Note: By default, system displays the Project Reference No. as search criteria

13. Click Search. System filters and displays the project on the basis of search applied.

14. By clicking on Clear, system resets the search criteria.

15. When both filter and search is applied, system shows the list of projects based on combination of filter and search criteria entered.

16. User Created Folders: By clicking on New Folder, the below pop up is displayed:

17. Enter the name of the new folder and click on Create.

Note: By clicking on Cancel, the New Folder pop up gets closed and navigate back to RCC Admin Inbox.

18. System displays the new created folder in RCC Admin Inbox

19. Edit User Created Folder: By clicking on Edit, the Edit folder pop up is displayed where you can edit the name of the folder.

20. By clicking Save, System saves the updated name in the RCC’s inbox.

Note:  By clicking on Cancel, the New Folder pop up gets closed and navigate back to RCC’s inbox

21. Delete User Created Folder: By clicking on Delete, System verifies if there are any projects existing in the folder which is getting deleted, if there are no projects in the folder, System asks for the confirmation with Delete and Cancel Button.

22. If there are projects in the folder, system displays a message “This folder contains project(s). To delete this folder, please move its project(s) to some other folder.

25. Move projects between folders: You can drag and drop the projects from the below listed folders in the inbox.

  • System folders to any of the user created folders

  • From any of the user created folders to any other user created folder

  • From any of the user created folders to its original system folder.

 Right Hand Side section of RCC Admin’s Inbox

1. You can click on any column of the project except project reference number hyperlink to view the right hand side section of the project.

2. Project Reference Number: RHS displays project reference number as hyperlink. You can click on Project Reference Number hyperlink to open complete project in a new browser/tab.

3. If you roll over the mouse on project reference number, the system will display the “project details” as added on project information tab by requestor.

4. Project Status: Project status is displayed in brackets next to project reference number hyperlink

5. Actions: The system displays the actions which can be performed on pending requests. The action buttons are enabled for the project depending on the orphan or high priority request.

6. The system displays the resource details under two sections:

  • Resources: The resources added on resource tab are displayed under resources section

  • Resources of Existing Releases: The tracks of existing releases which are a part of the project are displayed under this tab.

7. Manage Contract: By clicking on Manage, system displays the manage contract pop-up with the associated primary artist(s) linked with the resource(s).

8. You can search for any other contract for different artist(s) in GRS based on below criteria:

 

9. Select the Contract(s) and click on Link to Resource.

10. Select one or more of the contracts displayed on manage contract pop up and click on Save. System navigates back to the Inbox RHS with the new contract linked with the resource displayed.

11. Create Clearance Routing Contract: Click Create Contract, System displays the following fields to create contract.

12. Click Search Company, the Manage company popup is displayed as displayed below:

13. Enter the relevant search fields and click Search.

Note: By clicking on Reset, system clears the entered fields.

14. Select the company from the search result and click on Add.

Note: By clicking on Cancel, system exits the manage company pop up.

15. Click Search Artist button, the manage Artist pop up is displayed.

16. Enter the relevant search fields and click Search.

17. Select the Artist from the search result and click on Add.

18. You can enter the Clearance Notes in the text area and click on Create.

19. System displays the created clearance routing contract on manage contract screen. Click Save.

20. System will navigate back to Inbox RHS with the select contract displayed in contract column

21. Route: Select one or more request from the Admin Inbox and click Route.

22. System identifies the relevant Work group to route the request.

23. Approve: By clicking Approve hyperlink, system updates the final status of each of the request in project as approved and sends the notification to Requestor.

Note: Approve hyperlink is enabled only for One stop Request

24. Refresh: By clicking on the Refresh icon, you can manually refresh the Inbox at any point in time.

25. Notifications: The Inbox will have a notification icon with a counter to show the number of unread notifications in the Inbox.

26. Click on specific notification, System will directly navigate the requestor to the project in the Inbox in a new browser tab for which the notification has been received and right hand side of the project will be displayed.

27. System also removes any other notifications for the same project for all the users in the Inbox. It also removes the bold highlight against the project in Inbox for the individual user.

28. If you click directly on the project in the Inbox without clicking on the project notification, System removes all the notifications related to the project from the Inbox of the workgroup.
