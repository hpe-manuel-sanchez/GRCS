User Preferences

* A user in Requestor and Reviewer workgroup can maintain preferences for default settings and email notifications

1. Login to GCS and the home page of the system is displayed.

2. Click on User Preferences which is placed at the right top most header of the GCS page.

5. Inbox Role: This section is displayed on the screen if the user belongs to both, requestor and reviewer (UMGI marketing/UMGI global clearance/international legal/national legal/international marketing/national marketing / Local Label) roles. If the user belong to RCC Admin role also, this section is not displayed

6. You can select either requestor or reviewer. Only one option can be selected.

7. System will always display the default Inbox for the role selected in user preferences and user can manually switch to the Inbox for other role.

8. Requesting Company: This section is displayed if you belong to a workgroup with the requestor role

9. System displays the companies linked with the workgroup(s) for which the user has requestor role in the dropdown..

10. You can select one company from the dropdown.

11. System always displays the selected company as the requesting company by default when the user is creating a project in the clearance system.

12. Currency: This section is displayed if the user belongs to a workgroup with the requestor role.

13. You can select one currency from the dropdown.

14. System will always display the selected currency as the default currency on the project creation screen.

15. Receive a single consolidated mail for all the requests in a day: This option is applicable to you if you have a requestor and/or reviewer role.

Note: The option gets enabled on the screen only when the user has selected at-least one of the other options from the requestor/reviewer section.

16. If you select this option, system will send a consolidated email to you for all the notifications of the day (all the notifications which are selected in the user preferences screen). System will send a consolidate email at 7 AM CET and 7 PM CET.

17. Email preference options for Reviewer: The following email preferences are applicable if you have reviewer role in GCS

  • Receive Email for new requests in inbox: If this option is selected, system will send an email for all the new project notifications

  • Receive Email for requests removed from inbox because of cancellation: System sends email to the reviewer for all the requests which have been removed from the inbox because of project/resource cancellation

  • Receive Email for requests removed from artist consent folder because of cancellation : System sends email to the reviewer for all the requests which were pending in the artist consent folder and have been removed because of project/resource cancellation for the selected workgroups.

  • Receive Email for included/excluded requests in the master projects: The clearance system sends email to the reviewer for all the requests which were approved/conditionally approved by the reviewer but have been marked as included/excluded by the requestor

  • Receive Email for requests removed from inbox because of automatic project completion:

18. If you belong to more than one reviewer workgroup, the clearance system enables the list of workgroups along with roles  for which the you have reviewer role in the dropdown

Notes:

1. If the reviewer belongs to only one reviewer workgroup, then the workgroup dropdown is not displayed.

2. By default, all the workgroups will be selected by the system

19. You can de- select one or more workgroup(s) from the dropdown for which you do not want to receive the mail.

Note: If you have deselected all the workgroups, the clearance system displays a message to the user “please select at-least one workgroup to enable the preference”.

20. Email preference options for Requestor: The following email preferences are applicable if you have requestor role in GCS

  • Receive email for final review of Request: System sends email to the requestor whenever the request has completed the last level of review to show the final status.

  • Receive email when the reviewer has updated artist management flag : System sends email to the requestor for all the requests which have been marked by the reviewer for artist consent.
