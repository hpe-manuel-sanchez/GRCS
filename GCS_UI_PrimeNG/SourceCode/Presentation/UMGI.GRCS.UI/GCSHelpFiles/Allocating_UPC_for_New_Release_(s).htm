Allocating UPC for New Release (s)

UPCs can be allocated by Requestor and RCC Admin.

  • Allocation of UPC by Requestor

  • Allocation of UPC by RCC Admin

Allocation of UPC by Requestor

* Requestor should have UPC allocation rights in the work group.

1. Login to GCS and the home page of the system is displayed.

2. Navigate to the Requestor Inbox and click on the the project reference number of  Regular/Non-Traditional project to open the project in new browser/tab.

3. Navigate to the release tab.

4. System displays the Allocate UPC button at the bottom of the screen.

Note:

System verifies the following conditions to enable the UPC button:

1. The music type is NOT classical.

2. Multi-artist flag is selected on the project information tab.

3. The release does not hold a UPC.

4. The rights for UPC allocation have been assigned to the requestor.

5. The status of the project is submitted/re-submitted /re-opened.

5. Select one or more releases for which the UPC needs to be allocated.

6. Click Allocate UPC. System displays the generated 14 digit UPC against the UPC label for the relevant new release on the release tab.

Note: System disables the check box to set the flag to select the release(s) once the UPC has been allocated to all the release (s) added to the project.

When Requestor requests RCC admin to allocate UPC through the system.

Note: When Requestor does not have rights to allocate UPC or there is some issue, then Requestor request the RCC Admin to allocate UPC through the system.

7. Navigate to Emails >> UPC Allocation.

8. System sends the email notification to the RCC admin workgroup to allocate UPC.

UPC Allocation by RCC Admin

9. RCC admin can open the project for UPC allocation in two ways :

  • Through the RCC inbox (if project is available) : Click on the project reference number as hyperlink. System opens the project in a new browser and navigates to release tab.

  • By navigating to Admin>>UPC allocation: Search for project based on project reference number and click on the hyperlink. System will open the project in new browser /tab.

10. RCC admin can allocate UPC via to process:

  • Auto

  • Manual

UPC Allocation via Auto Process

For reference, please refer to Allocation of UPC by Requestor

UPC Allocation via Manual Process

11. Navigate to the Release tab.

12. System enables the editable text field against the UPC label on the New release screen for RCC Admin.

13. The RCC admin enters the 13 digit UPC manually in the first text field and the clearance system calculates and updates the 14th digit of UPC.

Note:

1. System will not allow the RCC admin to enter more than 13 digits of the UPC.

2. System will automatically calculate the 14th digit when the RCC admin has entered all the 13 digits and clicked anywhere else on the screen.

3. If the RCC admin has entered less than 13 digits, the clearance system will not calculate the 14th digit.

15. Once the UPC allocation is done, System displays the “remove” link against each allocated UPC.

Note:

1. Remove action is only applicable for RCC admin.

2. System displays the remove link against allocated UPCs in RCC Admin’s screen even if the UPC is allocated by Requestor.

16. By clicking on Remove link, system removes the UPC allocated to the release and enables the Allocate UPC button.

Note: System displays the “allocate UPC” button when there is at-least one new release without a UPC on the release tab.

17. System disables the “remove” link when the first push to R2 has been done.

Note: The remove link will remain enabled for manually allocated UPCs even after push to R2.
